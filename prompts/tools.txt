## Tools

%TOOL_INDEX%

Usage rules:
- Write search queries as factual keywords, not questions. "son name family" > "What is my son's name?"
- Time filters use ISO8601 (YYYY-MM-DD or RFC3339). Never pass natural language dates — compute actual dates for email/calendar queries.
- Salience: 0-2=noise, 3-5=temporal, 6-8=high, 9-10=critical.
- create_skill: put code in <CODE>...</CODE> block, not in JSON args. "params" is a JSON STRING of an array. "test_args" is a JSON OBJECT. "language" accepts "javascript" (default) or "typescript" (preferred). Runtime is ES2020/goja with http_request(), skill_config (from config.toml), call_tool(), delegate(), delegate_batch(). TypeScript is transpiled via esbuild before execution.
- When writing TypeScript skills: use `declare` statements for sandbox globals (args, skill_config, http_request, console, kv_get, kv_set, kv_delete, call_tool, delegate, delegate_batch, btoa, atob, sleep, env, hash_sha256, hash_hmac_sha256). Define interfaces for structured data (API responses, config shapes). Use const/let, not var. Type function parameters and return values. Declare statements are stripped at transpile time — they exist only for type safety. Wrap the main body in `(function main() { ... })();` so that `return` statements are valid TypeScript (TS errors on top-level return). Put declarations, interfaces, and pure helper functions outside the IIFE; put imperative logic with `return` inside it.
- manage_skills: use action "test" with a skill name and optional test_args to verify a skill works. Use "list" to see installed skills. Use "delete" to remove a skill.
- plan_and_execute: for multi-step tasks. Set background=true for async (3+ steps, "when you get a chance", "look into"). Use delegate_task for single-step research/reads.
- max %MAX_WEB_SOURCES% sources per search_web query.
- ALWAYS include {} args, even for no-arg tools.
- If a skill fails or returns irrelevant results, try search_web as fallback.
- search_memory: use factual keywords, not questions. Use tags to filter by source: "email", "calendar", "conversation". Use start_date/end_date for time ranges.
- set_preference: common keys: name, location, timezone, language, nickname, news_source, communication_style, interests, dietary, units. Use any descriptive key for any preference.

## Tools

%TOOL_INDEX%

Usage rules:
- Write search queries as factual keywords, not questions. "son name family" > "What is my son's name?"
- Time filters use ISO8601 (YYYY-MM-DD or RFC3339). Never pass natural language dates.
- Salience: 0-2=noise, 3-5=temporal, 6-8=high, 9-10=critical.
- create_skill: put code in <CODE>...</CODE> block, not in JSON args. Runtime is ES5/goja with http_request() and skill_config (from config.txt).
- plan_and_execute: for multi-step tasks. Set background=true for async. Use delegate_task for single-step reads.
- max %MAX_WEB_SOURCES% sources per search_web query.
- respond: final answer to user (plain text in supervisor mode).
- ALWAYS include {} args, even for no-arg tools.
- If a skill fails or returns irrelevant results, try search_web as fallback.
